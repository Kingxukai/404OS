1.主引导记录（MBR）：存储了磁盘分区表及开机启动程序。磁盘在引导时，BIOS 会加载 MBR 中的程序到内存（第 0 磁头、第 0 磁道、第 1 扇区），并将控制权转交给它。在 MBR 中，最后两位字节通常是 0xAA55，它们用来表示这是一个引导分区表。

2.保留区（Reserved Area）：紧接着 MBR 后面的区域称为保留区（reserved area），有时也称为引导扇区（bootstrap sector）。它包括了文件系统的引导扇区、备份引导扇区、FAT 表、FSINFO 扇区等。

    ·引导扇区（Bootstrap sector）：通常包括了操作系统所需的启动程序用来加载文件系统和驱动程序等操作系统必须的组件。引导扇区的大小通常是一个扇区或多个扇区。

    ·备份引导扇区（Backup Boot Sector）：是对引导扇区的一份备份，位于保留区的第 6 个扇区。

    ·FAT 表（File Allocation Table）：存储文件的分配信息，管理文件系统中哪些簇已经被分配。一般情况下，FAT 表只有 1 个或者 2 个备份，分别位于保留区的后续扇区。FAT 表的每个项标记了对应的簇是否已经被占据，以及该簇下一个簇地址（这样就形成一个链表）。

    ·FSINFO 扇区：是一系列与 FAT32 文件系统有关的关键信息所在的 1 个仿品段。这个扇区包含了重要的元数据信息，比如上一次文件系统检查的时间、总簇数、每簇扇区数等。
    
3.根目录区（Root Directory Area）：根目录区包含了文件系统的根目录下的所有文件和子目录的信息。在 FAT 文件系统中，根目录可以看作一个特殊的文件夹，这个文件夹会在分区格式化时就被预设好。它通常是一个固定大小，取决于簇大小和分区大小。

FAT32 文件系统采用簇的方式进行磁盘空间的分配，其簇中包含的扇区个数可以通过引导扇区的 BPB（BIOS Parameter Block）信息得知。

4.数据区（Data Area）：也称为文件簇区域，是存储文件数据的区域。数据区的容量由文件系统所使用的簇大小和簇数量所决定。
